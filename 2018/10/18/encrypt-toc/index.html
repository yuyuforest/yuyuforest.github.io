Hexo | 在NexT主题下加密文章和目录

发表于 2018-10-18 | 更新于 2018-11-19 | 分类于 探索

前言

折腾加密博文的起因是，我想把一篇很长的实验报告放到个人的github博客上，利用导航为TA提供阅读方便。当然，也有玩儿的意思在。但是我不想把我的隐私（学校班级、手机号等）暴露出来。之前建博客时，我看到别人的博客里提到了 hexo-blog-encrypt 这个插件，于是就开始尝试了。

（我知道可以在公布在博客上的报告里去掉那些隐私信息！我就是想搞搞加密嘛！）

这篇文章主要是为了记录配置加密TOC目录时的波折与思考，但是在讲这部分过程之前，不妨先说说怎么配置加密博文。

安装 hexo-blog-encrypt 插件

加密文章

参考源文档 hexo-blog-encrypt ，加密方法为在文章的md文件头，加上 password abstract message 等三个字段。

其中，password 无疑是表示文章的密码，abstract 表示你的文章在首页显示时的摘要（加密文章后，一般文章生成摘要所用的 <!--more--> 并不起作用），message 表示进入你的文章页面后，显示的消息语句。点击该语句即出现密码输入框。

保存文章后 hexo d -g ，看看效果如何。

我们会看到，进入文章页面后，侧边栏并不会像平时一样自动展开。手动展开它，并没有TOC。

我推测，这是因为没有对TOC的加密措施，为了保护文章内容，hexo-blog-encrypt 直接取消了目录的显示。

这可不是我想要的：）

那么，可不可以加密目录呢？

Bug ! 文章目录与站点概览同时出现

真不幸！出现了一个bug：

  • 进入文章页面后，侧边栏的导航处于“文章目录”，但同时显示了文章目录区与站点概览区
  • 将导航切换到“站点概览”，看到正常的站点概览
  • 再切换回“文章目录”，也是正常的文章目录

难道每次访问加密文章，都要手动点两次导航吗？？？这真是逼死个强迫症嘞：）

修复Bug

那么，来试试修好它吧！

检查网页

Ctrl + Shift + I ，或者右键点击网页 检查 ，进入开发者工具，在这里可以查看网页的 html 元素和样式等。

我们找到文章目录区与站点概览区对应的元素：

来切换一下两个区，看看元素的变化：

通过切换，可以发现，display 属性，会影响文章目录区/站点概览区的显示。

文章目录区和站点概览区里都有 sidebar-panel 和 sidebar-panel-active 这两个类， sidebar-panel 类里设置了 display: none ， sidebar-panel-active 里设置了 display: block 。

这个bug是站点概览区显示的错误，和这两个类有没有关系呢？
